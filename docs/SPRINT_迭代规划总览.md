# 养Young App 迭代规划总览

> 💰 **重要**: 关于付费功能使用规则，请查看 [付费操作确认规则.md](./付费操作确认规则.md)

---

## 迭代策略说明

### 核心原则
1. **优先开发管理后台**,提供可视化数据管理工具
2. 前端页面与后端 API 同步开发
3. 付费功能（登录验证码、AI API、文件上传等）延后实现
4. 使用管理后台维护种子数据,替代手动操作数据库
5. 使用默认资源（头像、图片等）避免早期文件上传

### 分阶段目标
- **第一阶段 (Sprint 0-0.5)**: 基础设施 + 管理后台 v1.0
- **第二阶段 (Sprint 1-8)**: iOS App 核心功能开发（使用后台管理数据）
- **第三阶段 (Sprint 9-10)**: 功能完善与优化
- **第四阶段 (Sprint 11-12)**: 集成付费功能（登录、AI、文件上传）
- **第五阶段 (Sprint 13)**: 测试优化与发布
- **第六阶段 (Sprint 14)**: 官方网站开发 🆕
