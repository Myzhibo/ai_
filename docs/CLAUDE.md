# Claude Code 工作规则

## 🎯 核心规则

### 1. 输出规范

- ❌ **不要**在每次操作结束后输出总结文档

### 2. Git 操作规范

- ❌ **不要**自行执行 `git add` `git commit` `git push` 等命令
- ✅ 所有 Git 推送操作由我自己执行
- ℹ️ 可以查看 git 状态和差异，但不要提交

### 3. 付费操作规范

- ⚠️ **必读**：详见 `付费操作确认规则.md`
- 🚨 所有付费操作必须停下来询问我
- 包括但不限于：
  - OpenAI/Claude API 调用
  - OSS/S3 文件存储
  - 短信验证码服务
  - 向量数据库付费套餐
  - 其他任何产生费用的服务

### 4. 整个项目完成过程中要思考的点

- 当前任务是否有相关AI Agent开发有关的知识点，如果有的话停下告诉我，询问我是否添加
  - 如： 当前任务可拆解独属于我的skills，然后添加到我的skills库中
  - 如： 当前任务可调用mcp工具，然后添加到我的mcp工具库中
  - 如： 当前任务可封装成独属于我的mcp工具，然后添加到我的mcp工具库中
  - ...

---

## 🔄 工作流程

1. 理解需求
2. 查看相关文档
3. 执行开发任务
4. 执行每个sprint任务前，告诉我当前sprint的任务清单，拆解单独任务后，告诉我每个任务是做什么的，然后依次执行任务，每个任务完成后，告诉我如何验证任务是否完成，然后我同意后再继续开发下一步任务
5. 一个sprint任务完成后，告诉我当前sprint任务是否完成，然后我同意后再继续开发下一个sprint任务

---

## ⚠️ 重要提醒

- 💰 涉及付费服务时，立即停止并询问
- 📝 不生成不必要的总结文档
- 🔧 不执行 Git 提交和推送操作
